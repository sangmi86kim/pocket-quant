# PocketQuant — 최적화 설계 노트

> PocketQuant의 "전략 탐색"을 최적화 문제로 정식화한 문서.
> 게임 컨셉(포켓퀀트/체육관) 뒤에 실제로 어떤 최적화가 돌고 있는지,
> 그리고 시즌 v3 이후 어떤 운영 정책으로 넘어갈지 정리한다.
>
> v0.x~v2(가짜 데이터·단일목적 GA·v1 가중치 탐색·v2 4아레나) 시절 설계는 폐기·요약했다.
> 현행은 **2026-06-23 시즌 v3**(비용 모델 완성 + 3교실 학습 + 국면별 리그, §4)이며,
> 다음 단계는 **시즌4 Regime Scanner 오버레이**(§5)다.

## 1. 목적함수 설계에서 발견한 함정 (전부 실측, 재발 방지용)

최적화기는 목적함수의 빈틈을 반드시 찾아낸다. 지금까지 "아무것도 안 하기(전부 현금)"가
최적해로 부활한 경로가 세 번 있었다:

| # | 경로 | 증상 (실측) | 봉인 방법 |
|---|------|------------|----------|
| 1 | **HP(현금 비중)를 적합도에 포함** | 전부 현금이 HP 100 + DEF 만점 = 69점 1위 | HP 가중치 0 (표시 전용), DEF=Calmar로 재설계 |
| 2 | **worst-case(min) 가중치 과다** | 현금은 전 체육관 균일 16.7점 = "최약 과목이 없음" → min 가중 0.4부터 현금 30~43/65위 부상 | min 가중치 0.3 (게이트를 지키는 실측 최대치) |
| 3 | **score_vs_dca를 평균으로 결합** | 하락 체육관에서 DCA가 마이너스 → 현금이 이김 → 평균 시 현금 24~53/65위 | 평균 결합 금지 — 비교 벡터는 해설·진단용으로 유지 |

시즌 v3에서 추가로 확정한 함정:

| # | 경로 | 증상 (실측) | 봉인 방법 |
|---|------|------------|----------|
| 4 | **비용 모델 반쪽 상태로 학습** | 수수료만 있고 슬리피지·No-trade band가 없어 고회전 전략 우위가 과대평가됨 | `cost_model.complete == True` 산출물만 소비. 기본 모델 `season3_flat_1bp_band5` |
| 5 | **turnover를 돈 버는 목적처럼 최소화** | 주식 비중 하루 차이가 작다는 사실이 수익 직관과 직접 연결되지 않음 | turnover는 목적함수에서 제거. 운용 스펙(`turnover_cap`)과 cap sweep으로 자른다 |
| 6 | **모든 국면을 한 트레이더에게 강제로 보충** | 보충수업이 약점 보완 대신 강점 훼손으로 나타남(NSGA/CMA-ES 보충 < 1차) | 잘하는 국면을 쓰고, 못하는 국면은 Regime Scanner가 저축왕/대체 포지션으로 회피 |

- 회귀 가드: `tools/test_baselines.py` (전부 현금 < 풀매수 & 하위 25% 룰).
- 공통 교훈: **0~100 클램프 스탯/BST를 최적화 목적으로 쓰지 말 것** (raw 지표 사용).
  HP가 아니라도, "낮은 노출 = 안정 점수 공짜"가 되는 모든 통로가 같은 퇴화를 만든다.

## 2. 일반 다목적 최적화 파이프라인과의 매핑

이 랩의 구조는 산업 현장의 전형적인 다목적 파라미터 튜닝 파이프라인 그대로다:

| 일반 최적화 파이프라인 | PocketQuant v3 |
|---|---|
| 품질 지표 N개 (목적함수) | NSGA-III 2목적: 중앙값 종료잔고·최악 종료잔고 |
| 운용 제약 | turnover cap, 비용 모델 완성도, No-trade band |
| 파라미터 탐색공간 | 시그널 가중치 len(SIGNAL_NAMES) + 후보 코덱 파라미터 |
| 더미 입력으로 루프 검증 → 실계측 교체 | 합성 아카데미 학습 → 실QQQ 졸업시험·리그 검증 |
| Pareto front → 운영 세팅 라인업 | front/topk → 교실별 1차·보충 라인업 |
| 검증 분리 | 학습 산출물 → 빅토리 로드 (OOS) 11년 → 사천왕 hold-out → 일별 국면 비교 |
| 운영 진단 | 전체 박스플랏 + 빅토리 로드 (OOS)/hold-out 분리 + bull/bear/sideways/volatile 박스플랏 |
| 콜백 훅 | Optuna callbacks (HV 조기종료) |

구조는 같고 도메인만 다르다 — **"목적함수의 빈틈을 옵티마이저가 먼저 찾는다"**는
교훈(§1)은 그대로 적용된다.

## 3. 한 줄 요약

```text
이기려는 대상 : 성실이(DCA), 저축왕, 어플삭제단 300명
목적          : 잘하는 국면에서 확실히 벌고, 못하는 국면은 운영 정책으로 피한다
학습          : CMA-ES · GP · NSGA-III 3교실 (TPE는 시즌3 본학습 벤치)
NSGA 목적     : [중앙값 종료잔고, 최악 종료잔고] maximize

검증          : 빅토리 로드 (OOS) 11년 + 사천왕 hold-out + 일별 국면 박스플랏
시즌4         : Regime Scanner가 bull/bear/sideways/volatile별로 출전·저축·줍줍을 결정
```

## 4. 시즌 v3 (2026-06-23 마감) — 비용 모델 완성 + 국면별 리그

시즌 v3의 핵심 변경은 **비용 모델을 실제 체결 모델 쪽으로 당긴 것**이다.
수수료만 과금하던 v2 이전 학습은 슬리피지·No-trade band가 빠져 고회전 전략이 과대평가될 수 있었다.
따라서 시즌 v3부터는 `app.pocket.battle.cost_model_metadata()["complete"] == True`인 산출물만
졸업시험·리그·보고서 입력으로 쓴다.

기본 비용 모델:

```text
cost_model      = season3_flat_1bp_band5
TRADE_COST      = 0.001   # 트레이더 수수료 0.1%/편도
SLIPPAGE_COST   = 0.0001  # 성실이 포함 전원 공통 1bp
NO_TRADE_BAND   = 0.05    # 목표 비중이 5% 이상 벗어날 때만 실제 체결
```

수수료는 전략 0.1%/편도, 성실이(DCA) 0원이다. 슬리피지는 성실이 포함 전원 공통이며,
No-trade band는 목표 포지션이 아니라 실제 체결 포지션과 turnover를 바꾸는 실행 모델이다.

### 4-1. 교실 라인업

TPE는 CMA-ES 대비 뚜렷한 우위가 없어 시즌3 본학습에서 벤치로 보냈다.
현행 학습 라인업은 3교실이다:

| 교실 | 역할 |
|---|---|
| CMA-ES | 단일목적. 중앙값 종료잔고 maximize |
| GP | 단일목적. seedleague 방식으로 시드별 best 5명 선발 |
| NSGA-III | 다목적. 중앙값 종료잔고 + 최악 종료잔고 동시 maximize |

1차 학습과 보충수업 산출물을 모두 보존하고, 시즌3 리그에서 둘을 별도 그룹으로 비교했다.

### 4-2. X — 결정변수

`app/academy/training/candidate.py`가 후보를 디코딩한다.
결정변수는 시그널 가중치 + 후보 코덱 파라미터이며, 현행 시그널 풀은 `SIGNAL_NAMES` 기준으로 평가한다.
탐색층은 교실별 sampler만 다르고, 채점 경로는 `app/academy/exam/grade.py`와
`app/pocket/battle.py`의 공용 전투 엔진을 탄다.

### 4-3. Y — 목적함수

NSGA-III는 2목적이다:

```text
Y = [ median_balance = 합성 체육관 중앙값 종료 잔고,  # maximize
      worst_balance  = 가장 운 나쁜 체육관 종료 잔고   # maximize
    ]
```

turnover는 돈 버는 목적이 아니라 운용 스펙이다. 따라서 목적함수에서 제거하고,
`turnover_cap` 필터와 `turnover_cap_sweep` 비교로 이동했다. "주식 비중 하루 차이가 작다"는 사실만으로
수익을 잘 낸다고 볼 수 없고, 반대로 하락장에서 사모으는 전략은 당장 아파도 다음 상승장 대비가 될 수 있다.

단일목적 교실(CMA-ES/GP)은 중앙값 종료잔고 maximize를 쓴다.

### 4-4. 선발

NSGA-III는 `graduate.select_topk`에서 졸업자를 median/worst 정규화 점수로 줄세운다.
`graduated` 태그는 리그 진출 게이트가 아니라 turnover cap 통과 여부를 나타내는 운용 태그다.

단일목적 교실은 점수순으로 선발한다:

| 교실 | 선발 |
|---|---|
| CMA-ES | 점수순 top30 |
| GP | seedleague 시드별 best 5명 |
| NSGA-III | median/worst 정규화 top30 |

시즌3 리그 출전자는 1차·보충을 모두 포함해 130명이다:

```text
CMA-ES-1차 30명 + CMA-ES-보충 30명
GP-1차 5명 + GP-보충 5명
NSGA-1차 30명 + NSGA-보충 30명
```

### 4-5. 리그 검증

시즌3 리그는 `app/league/v3/season3_league.py`에서 실행한다.
검증은 빅토리 로드 (OOS) 11년과 사천왕 hold-out을 분리하고, 일별 Regime Scanner 라벨로
`bull`/`bear`/`sideways`/`volatile` 국면별 박스플랏을 따로 그린다.

기준선은 성실이, 저축왕, 돼지저금통, 어플삭제단 300명이다.
어플삭제단은 랜덤 진입일 풀매수 300명 전체 분포를 사용해 min/max까지 확인한다.

정식 산출물:

```text

app/lab/reports/season/season3_league/season3_league.json
app/lab/reports/season/season3_league/graph/*.png
```

`app/lab/season/season3_bear_accumulation.py`는 하락장 매수분이 다음 상승장에서 얼마나 회복되는지 본다.

핵심 결과:

|---|---:|---:|

| CMA-ES-1차 | 23.6% | 0.46 |
| GP-보충 | 23.1% | 0.50 |

그래서 시즌4에서는 하락장을 무조건 회피로만 보지 않고, "줍줍 가능한 하락장"과 "현금 대기할 하락장"을
Regime Scanner가 더 잘 나눠야 한다.

## 5. 시즌4 Regime Scanner 오버레이

시즌4의 방향은 "한 트레이더가 모든 국면을 다 잘해야 한다"가 아니다.
**잘하는 국면에는 출전시키고, 모르면 저축왕으로 간다.**

현행 운영 가설:

| 국면 | 운영 후보 |
|---|---|
| bull | 공격 트레이더 출전. 시즌3 트레이더들이 가장 잘하는 구간 |
| bear | 무조건 도망이 아니라 줍줍 후보 검토. NSGA-1차/DCA형 누적 매수 분석 필요 |
| sideways | 저축왕 우선. 억지 거래보다 아무것도 하지 않는 편이 나을 수 있음 |
| volatile | 추가 분해 필요. 바닥/패닉, 휩쏘, 상승장 조정장을 같은 라벨로 묶으면 판단이 흐려짐 |
| unknown | 저축왕. 잘 모르겠으면 아무것도 하지 않는다 |

국면 라벨은 연 단위가 아니라 **일 단위**로 봐야 한다.
1년 단위로 자르면 대부분 상승장으로 뭉개져, 급락·횡보·변동 구간에서 실제로 어떤 일이 일어나는지
보이지 않는다.

시즌4 연구 항목:

1. Regime Scanner 고도화: `volatile`을 바닥/패닉, 휩쏘, 상승장 조정장으로 분리.
2. 하락장 대응 재해석: 당장 방어 점수와 다음 상승장 평단가 효과를 별도 목적/진단으로 분리.
3. 횡보장 정책: 저축왕 또는 초저회전 후보를 기본값으로 두고, 거래가 필요한 예외만 찾는다.
4. 보충수업 재설계: 약점을 억지로 보충하지 말고, 강점이 훼손되는지 먼저 확인한다.
5. 리그 리포트 기본값: 전체 비교와 국면별 비교를 항상 분리하고, 그래프는 matplotlib PNG로 남긴다.

## 6. hold-out 오염과 미래 봉인

post-COVID hold-out은 v2 재경기로 이미 오염됐다. 시즌3 리그의 사천왕 결과는
"이미 본 과거에 대한 참고 검증"이지, 앞으로 다시 깨끗한 최종 시험지가 될 수 없다.

현재 깨끗한 최종 판정지는 `FUTURE_SEAL_DATE = 2026-06-19` 이후 쌓이는 미래 데이터뿐이다.
미래 데이터를 개봉할 때는 확정된 모델 1개만 평가하고, 결과를 본 뒤 파라미터를 다시 고치지 않는다.
