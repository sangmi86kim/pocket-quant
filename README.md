# 🎮 PocketQuant

> **Gotta Money 'Em All**

포켓몬 리그 우승에 강한 포켓몬 하나만으로는 부족하다.
타입 상성, 역할 분담, 밸런스를 고려해 **최적의 파티**를 구성해야 한다.

투자도 마찬가지다.

PocketQuant는 **포켓몬 파티 구성을 포트폴리오 최적화 문제로 재해석**하여,
유전 알고리즘(GA, 나아가 NSGA-III)으로 **최적의 전략 조합을 탐색하는 실험 프로젝트**다.

> 📈 **실데이터(yfinance) 백테스트 기반입니다.** 각 전략을 실제 시장 국면(닷컴·금융위기·코로나·금리쇼크)의
> SPY 가격으로 백테스트해 **HP/ATK/DEF/SKILL 4스탯**을 뽑습니다. (투자 자문 아님 — 학습용)

## 🎯 이 프로젝트가 풀고 싶은 문제

최강의 포켓몬은 존재할 수 있다.
하지만 **모든 체육관을 이기는 포켓몬은 존재하지 않는다.**

투자 전략도 마찬가지다.

PocketQuant는 **"가장 강한 전략"** 이 아니라,
**"다양한 시장 국면에서 살아남는 전략 집합"** 을 찾기 위해 만들어졌다.

> 이 문제의식이 단일목적 GA를 넘어 **다목적 최적화(NSGA-III)** 로 향하는 이유다.
> (자세한 정식화는 **[OPTIMIZATION.md](OPTIMIZATION.md)** 참고.)

## 🧬 유전자 (시그널) — 3타입 × 2마리

전략은 아래 유전자를 조합해 만들어집니다. 각 유전자는 **실제 지표 로직**으로,
가격을 받아 그날의 포지션(0~1, 1=풀매수·0=현금) 또는 **기권**을 만듭니다.

| 타입 | 유전자 | 로직 (포지션) |
|------|--------|------|
| 💧 위험회피 | `DD`  | 드로다운 스탑 — 고점 대비 −10% 넘게 빠지면 현금화 |
| 💧 위험회피 | `VOL` | 실현변동성 레짐 — 평온=탑승 / 중간=반반 / 격동=현금 |
| 🔥 추세순응 | `MA`  | 가격 > 200일 이평이면 탑승 |
| 🔥 추세순응 | `MOM` | 모멘텀 — 최근 ~3개월 수익률이 양수면 탑승 |
| 🌿 역발상 | `REV_RSI` | **이벤트형** — RSI(14) < 30 과매도 투매면 매수 의견, 평소엔 기권 |
| 🌿 역발상 | `REV_BB`  | **이벤트형** — 볼린저 하단밴드 아래 과대 낙폭이면 매수 의견, 평소엔 기권 |

**전략 포지션 = 그날 의견 낸 유전자들의 평균 (기권 제외).** 이벤트형(REV)은 평소
벤치에 앉아 있다가 투매가 터진 날만 출전합니다 — "의견 없음"이 "현금 가라"로
집계되던 함정(잉어킹 강제 출전)을 막는 규칙. 전원 기권한 날은 현금(0)입니다.

> 🧪 구 `RSI`(과열 회피)·`BB`(상단 회피)는 실측 결과 평균 포지션 0.84/0.97 =
> 사실상 죽은 시그널이라 2026-06-10 재배치에서 제외했습니다
> (근거: [worklog](worklog/2026-06-10_signal_rework_plan.md)).

예시 이름: `MA-VOL킹`, `VOL-REV_BB몬`, `DD-MA-MOM몬`, `타이탄 드래곤`

## 💱 매매 방식 (어떻게 사고파나)

신호가 뜨면 "전부 사고/판다"가 아니라, **매일 "주식을 몇 % 들고 있을지(비중)"** 를 정해 거기에 맞춥니다.

- 각 유전자가 매일 비중을 뱉습니다: `1.0`=풀매수 · `0.0`=전부 현금 · `0.5`=반반 · 기권=의견 없음
- 전략 비중 = 그날 **의견 낸** 유전자들의 비중 **평균** (기권은 제외, 전원 기권이면 현금)
- 매일 그 비중으로 **리밸런싱** (비중이 그대로면 보유, 바뀌면 그만큼만 사고팜)
- 사고판 금액에는 **수수료 0.1%** (토스증권 미국주식 기준)를 차감

```
그날 수익 = (어제 비중) × (오늘 SPY 등락)
```

예) 어제 50% 들고 있었는데 오늘 SPY +2% → 내 자산 +1%. 현금 부분은 이자 0%.
신호는 종가로 확인하고 **다음 날** 반영합니다 (미래를 미리 안 보게 하루 lag = 룩어헤드 방지).

> ⚠️ **단순화된 시뮬레이션**입니다 — 수수료 0.1%는 반영하지만 세금·슬리피지·환율은 0,
> 매일 정확히 비중 조정, 소수점 주식 가능, 현금 이자 0% 가정. "어느 전략이 어느 시대에
> 강한가"를 비교하는 연구 도구이지 실거래 권유가 아닙니다.

## 🏟️ 체육관 (시장 국면)

각 체육관은 **실제 역사적 기간**을 가리키며, 그 기간의 SPY 가격으로 백테스트합니다.
(`difficulty`/`volatility`는 이제 판정엔 안 쓰는 **연출용 메타데이터** — 실제 난이도는 가격이 직접 만듭니다.)

| 체육관 | 기간 | 성격 |
|--------|------|------|
| 2008 금융위기 | 2008-01 ~ 2009-06 | 시스템 붕괴 대폭락 → 방어형 천국, 공격형 지옥 |
| 2020 코로나 급락 | 2020-02 ~ 2020-06 | V자 급락 후 즉시 회복 → 버티면 생존 |
| 2017 불타입 상승장 | 2017-01 ~ 2017-12 | 저변동성 꾸준한 우상향 → 추세추종/모멘텀 강세 |
| 2015-2016 횡보장 | 2015-01 ~ 2016-12 | 방향 없이 출렁임 → 추세형 헛신호, 평균회귀 유리 |

> 💡 폭락(2008)·급락(2020)·상승(2017)·횡보(2015-16) 4국면을 일부러 섞었다. 한 전략이 네 국면을 다 잘하긴 어렵다 — 이 충돌이 스탯에 그대로 찍히고, 다목적 최적화가 필요한 이유가 된다.

## ⚔️ 전투 로직 (백테스트 → 스탯)

```
1) 유전자들이 만든 일별 포지션(0~1)을 하루 lag 적용해 시장 수익에 곱함 (룩어헤드 방지)
   + 포지션 변화량 × 0.1%를 거래비용으로 차감 (토스증권 미국주식 위탁수수료 기준)
2) 체육관 기간(gym.start ~ gym.end)만 잘라 자산곡선에서 CAGR / 최대낙폭(MDD) / 샤프 / 평균현금을 측정
3) 0~100 스탯으로 정규화:
     ❤️ HP    = 평균 현금 비중           (위기 버틸 체력 — 표시 전용, 적합도 제외)
     ⚔️ ATK   = CAGR                      (0%→0, +25%→100)
     🛡️ DEF   = Calmar = CAGR/|내MDD|     (-1→0, 3→100 · 낙폭 한 단위당 수익)
     ✨ SKILL = 샤프비율                  (-1→0, 3→100)
```

종족치 `BST = HP+ATK+DEF+SKILL` (0~400). 등급은 적합도(**ATK/DEF/SKILL** 가중평균, 0~100)로:
`S`(≥90) `A`(≥70) `B`(≥50) `C`(≥30) `D`(그 외)

> 🧟 **"아무것도 안 하기" 퇴화 방지:** 예전 설계(HP 포함 동일가중 + DEF=시장 대비 낙폭비 +
> ATK -25%~+25% 스케일)에서는 '전부 현금' 전략이 HP 100 + DEF 100을 공짜로 받아 적합도
> ~69점으로 거의 모든 진짜 전략을 이겼다. 그래서 ① HP는 적합도에서 빼고(현금은 수단이지
> 목표가 아님) ② ATK는 0%부터 시작해 안 벌면 0점 ③ DEF는 비중 축소에 점수를 안 주는
> Calmar로 바꿨다. 재설계 후 '전부 현금'은 65마리 중 꼴찌(16.7점), '항상 풀매수'는 38.2점.
> 검증 스크립트: `python tests/test_baselines.py`

단, 전략을 종합 등급 하나로만 판별하지는 않습니다. 콘솔/Markdown 리포트에는
`종합·공격·방어·체력·효율` 관점별 등급과 `상승장 공격형`, `낙폭 방어형` 같은 스타일 판정도 함께 표시합니다.
예를 들어 연수익이 좋은 전략은 종합 C라도 공격 B로 분리해 해석합니다.

> 🪙 **"다 꼬라박아도 상폐만 아니면 진 게 아니다"** — 생존/사망 이진판정을 버리고
> 전부 연속 스탯으로 잰다. 페널티(0점 처리)는 없다.

### 데이터 기준

- `data.py`가 yfinance에서 수정종가를 받아 `data_cache/`에 저장합니다.
- `Gym.end`는 **포함 날짜**입니다. yfinance의 `end` 인자는 배타적이라 내부에서 하루를 더해 요청합니다.
- 캐시가 요청 종료일까지 충분하면 오프라인으로 그대로 쓰고, 종료일이 빠진 오래된 캐시는 다시 다운로드합니다.
- `battle.py`는 데이터 I/O 없이 이미 로딩된 가격만 계산하며, 평가 구간 밖 데이터는 스탯에 섞지 않습니다.

## 🚀 실행

> ⚠️ 반드시 **프로젝트 루트**에서 실행하세요. `app/backend/` 내부 파일을 직접 실행하면 상대 import가 깨집니다.
> 한글이 깨지면 (Windows): `$env:PYTHONIOENCODING="utf-8"`
> 의존성: `pip install -r requirements.txt` (yfinance·pandas·numpy). 첫 실행만 SPY 데이터를 받아 `data_cache/`에 캐시 → 이후 오프라인 동작.

실행 옵션은 전부 **config.json**으로 관리합니다 (CLI 플래그 없음 — 값 바꾸고 `python main.py`):

```jsonc
{
    "mode": "single",     // single(단판) | evolve(진화 GA) | dex(도감)
    "genes": null,        // [단판] 유전자 개수 (null = 랜덤)
    "pop": 20,            // [진화] 개체군 크기
    "generations": 10,    // [진화] 세대 수
    "seed": null,         // 랜덤 시드 (숫자 넣으면 재현 가능)
    "md": null,           // Markdown 리포트: null=안 씀, ""=기본 경로, "경로"=지정
    "capital": null       // 실전 시뮬 시작 자본(원). 예) 10000000
}
```

```bash
python main.py    # config.json의 mode대로 실행
```

| mode | 의미 | 주로 쓰는 키 |
|------|------|------|
| `"single"` | 단판 — 전략 한 마리 만들어 전 체육관 도전 | `genes`, `seed`, `md`, `capital` |
| `"evolve"` | 진화 — 단일목적 GA로 챔피언 탐색 | `pop`, `generations`, `seed`, `md`, `capital` |
| `"dex"` | 도감 — 유전자(포켓몬) 설명 카드 출력 | - |

#### 출력 예시

```
=== PocketQuant 진화 백테스트 ===
개체군 20마리 · 진화 10세대

1. 데이터 로딩: SPY 실데이터 4개 국면

[ 1세대] 최고 적합도  50.5점  최강 전략: REV_BB
[ 3세대] 최고 적합도  45.0점  최강 전략: VOL+REV_RSI+REV_BB
[ 7세대] 최고 적합도  54.8점  최강 전략: VOL+REV_BB
[10세대] 최고 적합도  54.8점  최강 전략: VOL+REV_BB

=== 최종 챔피언 ===
유전자: VOL, REV_BB
이름: VOL-REV_BB킹
최종 적합도: 54.8점 / 100

종합 스탯
  체력   HP    현금 비중    34.4점  #######
  공격력 ATK   연수익       49.0점  ##########
  방어력 DEF   낙폭대비수익    61.3점  ############
  솜씨   SKILL 샤프       54.1점  ###########

시장별 성적 (종족치 낮은 순 = 약한 시장):
  2015-2016 횡보장 체육관    83.6점  ####
  2008 금융위기 체육관        93.7점  #####
  2017 불타입 상승장 체육관    286.2점  ##############
  2020 코로나 급락 체육관     331.6점  #################
```

> 📌 **이 출력의 핵심:** 2026-06-10 시그널 재배치 후 챔피언은 `VOL+REV_BB`(54.8점)로
> 수렴합니다 — 전 조합(63개) 전수조사 1위와 일치하고, **방어 타입(VOL) + 역발상
> 타입(REV_BB)의 상성 조합**이라는 점이 재배치가 의도한 그림 그대로입니다. 재배치 전
> 챔피언(`VOL` 단일 40.0점) 대비 4개 체육관 전부 개선됐고, 특히 코로나 체육관에서
> +2.8% → **+12.2%** (시장 -3.2%) — 투매에 사는 REV_BB가 V자 회복을 잡은 결과입니다.
> 그래도 챔피언은 여전히 횡보장·금융위기에선 약합니다. "한 전략으로 모든 국면을 못
> 이긴다" — **다목적 최적화(NSGA-III)** 로 가는 이유입니다.

## 📁 구조

3층 구조: **main(입력) → service(흐름) → backend(계산)**

```
pocket_quant/
├─ main.py                 # config.json 읽어 service 호출 (CLI 플래그 없음)
├─ config.json             # 실행 옵션 (mode/genes/pop/generations/seed/md/capital)
├─ requirements.txt        # yfinance · pandas · numpy
├─ app/
│  ├─ service.py           # 실행 흐름 조립 (단판/진화/도감 순서 + 시드 + 출력)
│  └─ backend/             # 기능별 서브패키지
│     ├─ core/             #   models.py — dataclass + 스탯 가중치/등급
│     ├─ market/           #   data.py(가격+캐시) · gym.py(체육관 4종)
│     ├─ genes/            #   signals.py(유전자→포지션/기권, 결합 규칙) · dex.py(도감)
│     └─ engine/           #   battle.py(백테스트→스탯) · strategy.py(생성) · evolve.py(GA)
├─ tests/                  # 퇴화 검증 · 시그널 진단 · 워크 포워드
└─ worklog/                # 실험실 노트 (리뷰·계획서·검증 결과)
```

검증 도구 세 개를 상시 운용합니다:
- `python tests/test_baselines.py` — '전부 현금'/'항상 풀매수' 기준선을 전 조합과 줄세워 "아무것도 안 하기"가 상위권에 못 오는지 확인
- `python tests/check_signals.py` — 시그널 노출/발동률/상관 진단 (풀 교체 시 자유도 확인)
